# Android PR Review Skill

Compose Multiplatform (KMP + CMP) 프로젝트를 위한 PR 리뷰 스킬입니다.

## 리뷰 목적

코드 리뷰의 목적은 스타일 통일이 아니라 **안정성, 성능, 유지보수성 확보**에 있습니다.
과도한 추상화나 교과서적인 개선 제안보다는, **실제 운영 환경에서 문제가 될 수 있는 지점**을 우선적으로 검토합니다.

## 리뷰 우선순위

### High Priority (개별 라인 코멘트)

반드시 수정이 필요한 심각한 이슈입니다. PR 머지 전 해결이 필요합니다.

- **메모리 누수 위험**
  - 장수 객체가 Context, Activity, View 등을 직접 참조
  - Singleton/static 객체가 UI나 Context를 참조
  - 취소되지 않는 Coroutine Scope
  - `DisposableEffect`에서 onDispose 누락

- **크래시 위험**
  - 근거 없는 `!!` 사용
  - null, 캐스팅, 인덱스 접근에 대한 방어 코드 부재
  - catch 후 아무 처리도 하지 않는 코드
  - 공통 코드에서 플랫폼 전용 API 호출

- **멀티플랫폼 위반**
  - 공통 모듈에서 Android/iOS 플랫폼 API 직접 사용
  - expect/actual 없이 플랫폼 의존 코드 작성

### Medium Priority (전체 코멘트)

개선이 권장되는 이슈입니다. 가능하면 이번 PR에서 수정합니다.

- **성능 이슈**
  - 불필요한 리컴포지션 유발
  - mutable 객체를 Composable에 직접 전달
  - Composable 내부에서 객체/람다를 매번 새로 생성
  - Composable 내부에서 무거운 연산 수행

- **설계 이슈**
  - 하나의 클래스가 여러 역할을 수행
  - UI 컴포넌트가 로직과 플랫폼 제어를 직접 수행
  - Side-effect가 명시적이지 않음

### Low Priority (전체 코멘트)

참고 사항입니다. 다음 PR에서 개선해도 됩니다.

- **코드 품질**
  - `remember`로 문제를 감추는 구조
  - 깊은 상속 구조
  - 역할별 Composable 분리 미흡

- **유지보수성**
  - 코드 의도가 불명확
  - 불필요한 복잡성

## 리뷰 관점

1. **멀티플랫폼 환경에서 안전한가?**
2. **불필요한 리컴포지션을 유발하지 않는가?**
3. **크래시 가능성을 충분히 고려했는가?**
4. **책임이 명확하고 장기 유지보수가 가능한가?**

## 지양할 사항

- 개인 취향 기반의 스타일 지적
- 불필요한 아키텍처 제안
- 앱 전용 패턴을 SDK/공통 UI 코드에 강요
